# 가격 추정 모델 — 동적 지표 리서치 반영 노트

첨부 리서치(서울 버전 / 경기도 버전)를 추정 모델에 반영하면서 정리한 설계 메모.
**핵심 원칙**: 검증되지 않은 휴리스틱 가중치는 "추정 prior"로 라벨링하고, 자동 수집
불가능한 지표로 빈 값을 만들어 결과를 희석시키지 않는다.

## 1. 적용 방식 — 지역 레짐 스위칭 (regime switching)

서울 중심지와 경기·수도권 외곽은 상승 동인이 다르다. 그래서 **단지 주소로 레짐을
판별**하고, 이미 계산 가능한 가격 앵커들의 **가중치를 프로파일별로 재조정**한다.
(별도 momentum 항을 끼워넣지 않고, 기존 앵커가 expectedSaleMid 가중평균에
들어가므로 재조정만으로 예상가격이 움직인다.)

- 구현: `src/lib/priceModel.ts`
  - `regionProfileFromAddress(address)` → `"seoul" | "gyeonggi" | "default"`
  - `REGION_WEIGHT_MULTIPLIERS` — 프로파일별 가중치 배수 (⚠️ 휴리스틱 prior, 추후 보정)
  - `applyRegionProfile(weights, profile)` → 재조정된 가중치
- 호출: `src/app/targets/[id]/page.tsx` 에서 `apartment.address` 기반으로 주입

| 프로파일 | 강조(↑) | 약화(↓) | 근거 |
|---|---|---|---|
| seoul | 대장아파트 앵커, 상급지 압력, 호가 리딩 | 분양권 프리미엄 | 상급지 호가 리딩·견고한 인프라 선행 |
| gyeonggi | 분양권 프리미엄, 전세갭(하방가), 저가매물 소진 | 대장 앵커 | 풍선효과·실수요 매수전환·낙수효과 |

## 2. 리서치 지표 → 현재 모델 매핑 / 데이터 현실성

| 리서치 지표 | 상태 | 비고 |
|---|---|---|
| 매도 호가밴드 격차/정렬 | ✅ 반영 | 네이버 호가 (`saleListings`) |
| 전세가율·갭 | ✅ 반영 | `deriveJeonseRatio`, `jeonseFloorPrice` |
| 매물 소진속도 | ✅ 반영 | `lowPriceAbsorptionRate` |
| 대장아파트 신고가 리딩 | ✅ 반영 | `leaderTransactions` 앵커 |
| 분양권 프리미엄 | ✅ 반영 | 청약홈 분양가 + 분양권 실거래 |

## 3. 아직 자동 수집 안 되는 지표 — 향후 필요한 데이터/API (기록용)

리서치에는 있으나 현재 파이프라인으로 자동 수집 불가. 추후 확장 시 우선순위.

| 지표 | 데이터/API 후보 | 난이도 |
|---|---|---|
| 미분양 소진율(경기 ④) | 국토교통부 **미분양주택현황** 통계 (시군구 월별) — data.go.kr | 중 |
| 거래량 5개년 평균 돌파(경기 ⑤) | 국토부 아파트 실거래 누적 조회 후 자체 집계 | 중 (계산) |
| 외지인/법인 매수비중(경기 ⑤) | 한국부동산원 **매입자거주지별 거래현황** 통계 | 상 |
| 서울 동조화/상급지 price ratio(서울 ③, 경기 ①) | "상급지" 정의 + 상시 시세 파이프라인 필요 | 상 |
| GTX·신안산선·월판선 착공/개통 단계(경기 ②) | 구조화 API 없음 — 수기 입력 슬라이더로만 가능 | 상 |
| 시간차(Time-Lag) 모델 (상급지→호가→실거래) | 시세 시계열 저장 필요 | 상 |

→ 위 지표는 수집 인프라가 갖춰지면 `ModelWeights`에 항목을 추가하고
`REGION_WEIGHT_MULTIPLIERS`로 레짐별 가중을 부여하는 방식으로 확장한다.

## 3-1. 백테스트 리서치 반영 (2013~2024 실거래 다중회귀 — 출처 미검증 prior)

추가로 제시된 "백테스트 검증 가중치"의 **방향성**을 `REGION_WEIGHT_MULTIPLIERS`에
반영했다. 정확한 %는 독립 검증 불가하므로 방향만 채택하고 극단값은 회피.

| 요소 | 이론 | 백테스트(주장) | 모델 매핑 | 반영 |
|---|---|---|---|---|
| 서울 상급지 동조화/시차 갭 | 35% | 45% ▲ | comparableMarketPressure, leaderApartmentAnchor | 1.7 / 1.6 |
| 서울 교통 접근성 | 30% | 25% | (정적, locationPremium 일부) | — |
| 서울 전세가율 | 25% | 15% ▼ | jeonseFloorPrice | 0.75 (착시 보정) |
| 서울 대단지/브랜드/입지 | 20% | 10% ▼ | locationPremium | 0.8 (방어지표) |
| 경기 서울인접 격차 | 30% | 40% ▲ | comparableMarketPressure | 1.5 |
| 경기 교통망 단계 | 25% | 30% ▲ | (수기 입력 필요) | — |
| 경기 미분양 소진율 | 15% | 10% | inventorySignal + 미분양 API | 1.3 (proxy) |

### 백테스트 3대 법칙 (모델 반영 상태)
1. **시차 동조화** (강남→경기 3~6개월, R² 0.82 주장): 시계열 저장 필요 → **미반영**(향후).
2. **전세가율 착시** (불장에선 전세 비중↓): 서울 jeonseFloor 0.75로 부분 반영. 단,
   불장/하락기 자동 판별기가 없어 정적 적용 — **시장국면 감지 도입 시 동적화**.
3. **역세권/초품아 = 방어지표** (상승동인 아님): locationPremium 0.8로 하향 반영.

### 트리거(게이트) — 향후 1순위
경기 상승신호 활성화 조건: **해당 시군 미분양 < 5년 평균**일 때만 상승 가중 ON.
미분양 데이터(국토부 미분양주택현황 API) 확보 시, gyeonggi 프로파일의 상승계
가중(comparableMarketPressure 등)을 이 게이트로 조건부 활성화하도록 확장.

## 3-2. 시차(Time-Lag) 전이 모델 반영 (공급절벽 압축시차 리서치)

가격 요소가 상급지→당해지→임대차→실거래로 **시차를 두고 전이**된다는 리서치.
서울은 2~4주 초단기, 경기 외곽은 4~6주 지연 시차. 공급절벽 장세에선 더 압축.

| 가격 결정 요소 | 서울 시차 | 경기 외곽 시차 | 모델 반영 상태 |
|---|---|---|---|
| 상급지 동조화(갭 메우기) | 2~4주 | 4~6주 | 시계열 미보유 → **부분**(레짐 가중 + recency 차등) |
| 임대차(전세가율 압박) | 4주 이내 | 6~8주 | jeonseFloorPrice 가중(레짐별) |
| 교통/인프라 호재 | 상시 | 착공단계 계단식 | **미반영**(수기 슬라이더 필요) |
| 수급붕괴(입주물량 쇼크) | 즉시 잠김 | 4주 지연 | **공급절벽 모드**(수동 토글)로 근사 |

**실제 반영(코드):** `temporalWeight(date, { profile, supplyCliff })` — 시간감쇠 곡선을
레짐별로 차등화. 서울은 ≤1mo 1.5/≤3mo 1.3로 최근 신호를 강하게, 6mo 초과는 급감
(0.75→0.55). 경기는 1~3mo plateau를 넓혀(1.25) 선행 구간을 길게 인정, 완만 감쇠.
공급절벽 ON 시 ≤3mo ×1.15 추가 가중·12mo 초과 ×0.85 추가 감쇠로 시차 압축을 근사.

**한계(정직성):** 진짜 시차 모델(상급지 발화→당해지 전이의 선행 신호화)은 **상급지
시세 시계열·미분양 월별 통계·외지인 매수비중·GTX 착공단계** 데이터가 있어야 한다.
현재는 "최근 거래일수록 가중"이라는 recency 방향으로만 시차를 근사하며, 실제
선행지표 매핑은 §3 데이터 수집 인프라 구축 후 확장한다.

## 4. 주의 (객관성)

- 가중치 배수(1.4, 1.3 등)는 리서치 저자의 휴리스틱이며 **실증 계수가 아니다.**
  데이터가 쌓이면 백테스트로 보정해야 한다.
- 동적 지표는 본질적으로 "상승 압력(미래 기대)"이므로, 예상가격에 반영하더라도
  실측 앵커(실거래) 비중을 유지해 과도한 시세 선반영을 경계한다.
